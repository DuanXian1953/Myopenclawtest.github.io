📖 什么是 OpenClaw？

OpenClaw 是一个开源的 AI 助手框架，可以运行在你自己的设备上。它不是云端服务，而是完全本地运行的智能助手。

核心特点

  • 🏠 本地运行 - 在你的电脑上运行，数据不出设备
  • 🤖 多模型支持 - 支持 Kimi、通义千问、DeepSeek、Claude 等 30+ 模型
  • 💬 多渠道接入 - 支持飞书、Telegram、WhatsApp、Discord 等
  • 📁 文件访问 - 可以读写你的工作区文件
  • 🌐 网页浏览 - 可以搜索网络、访问网页
  • ⏰ 定时任务 - 支持 cron 定时任务和提醒
  • 🧠 记忆系统 - 长期记忆和日常笔记，保持上下文连贯
  • 📱 设备控制 - 可连接手机/平板，查看摄像头、屏幕等
💡 适合场景：个人助手、自动化任务、文件处理、日程管理、信息整理、代码辅助等

步骤 1：安装 Node.js

访问 nodejs.org 下载并安装 v22+ 版本。

步骤 2：安装 OpenClaw 
npm install -g openclaw
步骤 3：运行向导 
openclaw onboard --install-daemon

按提示选择 AI 模型（推荐 Kimi 或 DeepSeek）并完成配置。

步骤 4：开始对话 
openclaw chat

或打开浏览器访问 http://localhost:18789

📦 安装方法

步骤 1：安装 Node.js v22+

Windows：访问 nodejs.org 下载 .msi 安装程序。

macOS：

brew install node@22

Linux：

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
步骤 2：全局安装 OpenClaw 
npm install -g openclaw
步骤 3：验证安装 
openclaw --version
# 应输出版本号，如：2026.3.24
步骤 1：克隆仓库 
git clone https://github.com/openclaw/openclaw.git
cd openclaw
步骤 2：安装依赖并链接 
npm install
npm link
步骤 1：拉取镜像 
docker pull openclaw/openclaw:latest
步骤 2：运行容器 
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  -e OPENCLAW_MODEL=moonshot/kimi-k2.5 \
  openclaw/openclaw:latest

🎯 初始化与配置

运行交互式向导 
openclaw onboard --install-daemon

交互式向导将引导你完成以下配置：

  • 选择 AI 模型提供商 - Kimi、DeepSeek、通义千问、Ollama 等
  • 配置认证信息 - API 密钥或 OAuth 授权
  • 设置工作区目录 - 默认 ~/.openclaw/workspace
  • 配置渠道 - 飞书、Telegram、WhatsApp 等（可选）
  • 安装守护进程 - 开机自启动（可选）
💡 提示： 配置完成后会在 ~/.openclaw/ 目录生成配置文件

🤖 AI 模型配置

📁 配置文件位置：
• Windows: %USERPROFILE%\.openclaw\openclaw.json
• macOS/Linux: ~/.openclaw/openclaw.json

云端模型无需本地 GPU，通过 API 调用，适合大多数用户：

DeepSeek 配置

获取 API Key：访问 platform.deepseek.com

// ~/.openclaw/openclaw.json
{
  "auth": {
    "profiles": {
      "deepseek:default": {
        "provider": "deepseek",
        "mode": "api_key"
      }
    }
  },
  "models": {
    "providers": {
      "deepseek": {
        "apiKey": "sk-xxxxxxxxxxxxxxxx"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "deepseek/deepseek-chat"
      }
    }
  }
}

本地模型需要 GPU（建议 8GB+ 显存），数据完全本地，适合隐私敏感场景：

Ollama + Qwen 配置

# macOS / Linux

# Windows: 访问 ollama.com 下载安装程序

3. 验证 Ollama 运行：

ollama run qwen2.5:7b "你好"

// ~/.openclaw/openclaw.json
{
  "models": {
    "providers": {
      "ollama": {
        "baseUrl": "http://localhost:11434/v1"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "ollama/qwen2.5:7b"
      }
    }
  }
}

5. 启动并测试：

openclaw gateway restart
openclaw chat

📱 接入飞书

⚠️ 版本要求：OpenClaw v2026.2.26+ | 需要飞书企业账号和开放平台管理员权限

📋 前置条件

  • OpenClaw 版本 ≥ v2026.2.26
  • 飞书企业账号（个人版功能受限）
  • 飞书开放平台管理员权限
  • 已完成的 OpenClaw 基础配置

🔧 飞书应用创建

步骤 1：访问飞书开放平台

前往 open.feishu.cn/app 并登录企业账号。

步骤 2：创建企业自建应用
  1. 点击「创建企业自建应用」
  2. 选择「机器人」类型
  3. 填写应用名称（如：OpenClaw Assistant）
  4. 上传应用图标（可选）
  5. 点击「创建」
步骤 3：获取凭证

在应用管理页面：

  • App ID： 显示为 cli_xxxxxxxxxxxxx 格式
  • App Secret： 点击「查看」并验证后获取
💡 位置：应用管理 → 凭证与基础信息 → App ID / App Secret

🔐 应用权限配置

在飞书开放平台「权限管理」页面，添加以下权限：

必需权限列表
im:message im:chat im:resource contact:contact:readonly docs:document.content:read wiki:wiki:readonly
📋 权限说明
im:message 发送和接收消息（必需）
im:chat 管理会话信息
im:resource 访问消息中的资源（图片、文件）
contact:contact:readonly 读取联系人信息（可选）
docs:document.content:read 读取飞书文档内容
wiki:wiki:readonly 读取知识库内容

📡 事件订阅

⚠️ 重要：必须选择 WebSocket 模式（长连接），无需配置回调 URL！
配置步骤
  1. 在开放平台开启「事件订阅」
  2. 选择 WebSocket 连接模式
  3. 订阅事件：im.message.receive_v1
  4. 配置加密密钥（记录保存，用于验证消息）
  5. 保存配置

⚙️ OpenClaw 端配置

使用交互式 CLI 
openclaw channels add feishu

按提示输入：

  • App ID
  • App Secret
  • 连接模式（选择 WebSocket）
编辑配置文件 
// ~/.openclaw/openclaw.json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_a94d372767385bb7",
      "appSecret": "你的 App Secret",
      "connectionMode": "websocket",
      "domain": "feishu",
      "groupPolicy": "open",
      "dmPolicy": "open",
      "allowFrom": ["*"]
    }
  },
  "plugins": {
    "entries": {
      "feishu": {
        "enabled": true
      }
    }
  }
}
使用环境变量 
# ~/.openclaw/.env
OPENCLAW_FEISHU_ENABLED=true
OPENCLAW_FEISHU_APP_ID=cli_a94d372767385bb7
OPENCLAW_FEISHU_APP_SECRET=你的 App Secret
OPENCLAW_FEISHU_CONNECTION_MODE=websocket

  • 命令测试：发送 /help 查看可用命令
✅ 如果收到回复，说明接入成功！

❓ 常见问题

安装后 openclaw 命令找不到？

原因：npm 全局目录不在 PATH 中。

# Windows: 将 npm prefix 目录添加到系统 PATH
# macOS / Linux
export PATH=$(npm config get prefix)/bin:$PATH

Windows 永久添加：

  1. 右键「此电脑」→ 属性 → 高级系统设置
  2. 点击「环境变量」
  3. 在「系统变量」中找到 Path，点击编辑
  4. 新建，添加 C:\Users\你的用户名\AppData\Roaming\npm
  5. 确定保存，重启终端
飞书机器人不回复消息？

检查清单：

  1. 确认应用已发布到飞书
  2. 检查事件订阅是否开启 WebSocket
  3. 确认 im.message.receive_v1 事件已订阅
  4. 检查权限是否已授予
  5. 查看 OpenClaw 日志：openclaw gateway logs
如何切换 AI 模型？ 
/model kimi
/model qwen
/model deepseek
工作区在哪里？
  • Windows： C:\Users\你的用户名\.openclaw\workspace
  • macOS： ~/.openclaw/workspace
  • Linux： ~/.openclaw/workspace
Gateway 启动失败，端口被占用？

方法 1：查找并关闭占用端口的进程

# Windows
netstat -ano | findstr :18789
taskkill /F /PID <进程 ID>

# macOS / Linux
lsof -i :18789
kill -9 <进程 ID>

方法 2：修改 OpenClaw 端口

# 编辑配置文件
# ~/.openclaw/openclaw.json
{
  "gateway": {
    "port": 18790  // 改为其他端口
  }
}
Ollama 连接失败？

检查步骤：

  1. 确认 Ollama 服务正在运行：ollama serve
  2. 测试 Ollama API：curl http://localhost:11434/api/tags
  3. 确认模型已下载：ollama list
  4. 检查防火墙是否阻止 11434 端口
  5. Mac 用户可能需要允许 Ollama 通过防火墙
API Key 无效或过期？

解决方法：

  1. 登录对应模型平台确认账户状态
  2. 检查 API Key 是否正确复制（无多余空格）
  3. 确认账户有足够余额/额度
  4. 如仍无效，重新生成 API Key 并更新配置
  5. 检查是否需要配置代理访问
如何查看和导出聊天记录？

查看会话列表：

openclaw sessions list

查看特定会话历史：

openclaw sessions history --session <session-key>

导出为文件：

openclaw sessions export --format json --output chat-history.json
内存占用过高怎么办？

优化建议：

  • 使用较小的模型（如 7B 而非 14B）
  • 定期清理旧会话：openclaw sessions cleanup
  • 减少同时运行的子代理数量
  • 使用 Ollama 时调整上下文长度
  • 考虑增加系统内存或使用 swap
